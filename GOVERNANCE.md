# Governance — кто что может менять

## Роли

### Project Manager (PM)
Работает из корневой директории `DND_Assistant/`.
Это единственный агент, который управляет контрактами и задачами.

### Sub-agents (Backend / iOS / Web)
Работают каждый в своём репозитории. Выполняют задачи, поставленные PM через тикеты.

---

## Правила доступа

### _contracts/ — ТОЛЬКО PM

| Действие | PM | Sub-agent |
|---|---|---|
| Создать тикет в `tickets/open/` | ✅ | ❌ |
| Удалить / переименовать тикет | ✅ | ❌ |
| Переместить тикет в `done/` | ✅ | ❌ |
| Изменить приоритет / описание тикета | ✅ | ❌ |
| Отметить чекбоксы `[x]` ТОЛЬКО в секции своего проекта | ✅ | ✅ (только свой `## iOS` / `## Web` / `## Backend`) |
| Добавить комментарий в тикет (append) | ✅ | ✅ |
| Редактировать ROADMAP.md | ✅ | ❌ |
| Редактировать GOVERNANCE.md | ✅ | ❌ |
| Редактировать AGENT_GUIDE.md | ✅ | ❌ |
| Создать ADR в `decisions/` | ✅ | ✅ (свой проект) |

### Код — ТОЛЬКО свой проект

| Действие | Backend agent | iOS agent | Web agent |
|---|---|---|---|
| Код в `DNDAssistantBackend/` | ✅ | ❌ | ❌ |
| Код в `DNDAssistantIos/` | ❌ | ✅ | ❌ |
| Код в `DNDAssistantWeb/` | ❌ | ❌ | ✅ |
| `swagger.json` | ✅ (write) | read only | read only |

---

## Что sub-agent ОБЯЗАН делать

1. **В начале сессии** — прочитать `_contracts/tickets/open/` и найти свои задачи
2. **Во время работы** — отмечать `[x]` свои чекбоксы по мере выполнения
3. **В конце сессии** — закоммитить и запушить код
4. **Если нашёл баг / проблему в чужом проекте** — добавить комментарий в тикет, НЕ пытаться чинить

## Что sub-agent НЕ МОЖЕТ делать

1. ❌ Создавать новые тикеты
2. ❌ Менять приоритеты
3. ❌ Перемещать тикеты между `open/` и `done/`
4. ❌ Менять ROADMAP, GOVERNANCE, AGENT_GUIDE
5. ❌ Редактировать чужой код
6. ❌ Менять описание или спеки в тикетах (только комментарий с пометкой)

---

## Формат комментария от sub-agent

Если агент хочет сообщить о проблеме или предложить изменение в тикете:

```markdown
<!-- AGENT COMMENT -->
**[backend-agent, 2026-03-29]:** Обнаружил что поле X нужно сделать nullable,
иначе миграция сломает существующие записи. Жду решения PM.
<!-- /AGENT COMMENT -->
```

PM увидит комментарий и примет решение.

## Git правила

- Каждый sub-agent пушит только в свой репозиторий
- Коммит-сообщения: `<type>: <description> (TICK-XXX)`
- Типы: `feat`, `fix`, `refactor`, `docs`, `chore`
- Push в `main` — допустим (пока нет code review процесса)
- Remote `origin` — GitHub, `gitea` — TrueNAS backup
- После push в origin — push в gitea тоже
